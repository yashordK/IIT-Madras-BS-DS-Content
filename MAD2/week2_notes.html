L1

Arrays, Collections & Higher-Order Functions

Arrays in JavaScript

Arrays are the most basic collection type. In JS, arrays are objects — typeof [] returns "object". They can hold mixed types.

const x = [1, "hello", true, null];  // mixed-type array — valid in JS
console.log(typeof x);        // "object"
console.log(x.length);        // 4
console.log(x[0]);            // 1 (zero-indexed)

Iterables & Iterators

  • Iterable: any object whose contents can be accessed sequentially
  • Iterator: the object that tracks position and has a next() function
  • Arrays, strings, Maps, Sets are all iterables. Plain numbers and single characters are not.

Higher-Order Functions (Functions that take Functions)

JS treats functions as first-class objects — you can pass them as arguments. This powers methods like map, filter, find, reduce.

const nums = [1, -2, 3, -4, 5];

// find — returns FIRST element matching condition
const first = nums.find(n => n > 0);        // 1

// filter — returns ALL matching elements as new array
const pos = nums.filter(n => n > 0);        // [1, 3, 5]

// map — transforms each element, returns new array
const signs = nums.map(n => n > 0 ? "+" : "-"); // ["+","-","+","-","+"]

// reduce — accumulates to a single value
const sum = nums.reduce((acc, i) => acc + i, 0); // 3

Generators (Preview)

Functions that can yield values one at a time, pausing between yields. They enable concurrency — two "active" functions cooperating. Used more in advanced scenarios.

function* counter() {
  yield 1;
  yield 2;
  yield 3;
}
const gen = counter();
gen.next().value; // 1
gen.next().value; // 2
L2

Iteration, Spread & Destructuring

All Three Loop Styles

const x = [10, "b", undefined, 30];

// Classic C-style — use let (i++ reassigns)
for (let i = 0; i < x.length; i++) { console.log(x[i]); }

// for...in — gives KEYS (indices as strings)
for (const i in x) { console.log(i, x[i]); } // "0" 10, "1" "b", etc.

// for...of — gives VALUES directly
for (const val of x) { console.log(val); }  // 10, "b", undefined, 30
for...in on arrays gives keys as strings ("0","1"...). Use for...of for array values.

Iterating Objects

const obj = {a: 1, b: "alpha", c: 321};

// Object.entries() gives [key, value] pairs
for (const [k, v] of Object.entries(obj)) {
  console.log(k, v);  // a 1 | b alpha | c 321
}
// Also: Object.keys(obj) and Object.values(obj)

Spread Operator (...)

const a = [1, 2, 3];
const b = [0, ...a, 4];  // [0, 1, 2, 3, 4] — spreads a into b
// Without spread: [0, [1,2,3], 4] — nested array

Array Sorting

const nums = [10, 2, 30, 4];
nums.sort();              // [10, 2, 30, 4] — sorts as STRINGS by default!
nums.sort((a, b) => a - b); // [2, 4, 10, 30] — correct numeric sort
// Comparator: negative → a first; positive → b first; 0 → equal
Default .sort() converts to strings first! Always pass a comparator for numbers.

Destructuring

// Array destructuring
const arr = [1, 2, 3];
const [a, b] = arr;  // a=1, b=2 (c=3 is ignored)

// Object destructuring
const obj = {x: 10, y: 20};
const {x, y} = obj;  // x=10, y=20

// Rename while destructuring
const {x: myX} = obj;  // myX=10

// In function parameters
function show({name, age}) { console.log(name, age); }
show({name: "Yash", age: 20});

Modules & Objects

What is a Module?

A module collects related functions, objects, or values into a reusable unit. Like numpy in Python. Benefits: reusability, encapsulation, namespace isolation.

ES6 Modules — export & import

// === constants.js ===
export const PI = 3.14159;
export const c = 3e8;          // speed of light
const sq = x => x * x;          // NOT exported — private to module
export default function energy(m) { return m * c * c; }

// === main.js ===
import energy from './constants.js';   // default import
import { PI, c } from './constants.js'; // named imports
import { c as speedOfLight } from './constants.js'; // rename
import * as consts from './constants.js'; // import everything
Export typeSyntaxImport syntax
Named exportexport const x = 1import { x } from '...'
Default exportexport default fnimport fn from '...'
Namespace—import * as mod from '...'
Use <script type="module"> in HTML to enable ES6 modules in the browser.

npm — Node Package Manager

  • Node.js is a command-line JS runtime (mainly backend)
  • npm manages packages/modules for both backend AND frontend
  • Bundlers (webpack, Vite) package npm modules for browser use

JavaScript Objects

// Object literal — property: value pairs
const xx = {
  a: 5,
  b: 10,
  add(x, y) { return x + y + this.a; }  // this = the object itself
};
xx.add(3, 4);   // 3 + 4 + 5 = 12

// Add properties dynamically
xx.f = function(x) { return this.a + x; };
xx.f(10);  // 15
this in a regular function → refers to the calling object (like Python's self). In arrow functions → this is inherited from the outer scope (lexical).

Getters, Setters, Prototypes & Classes

Getters & Setters

const user = {
  first: "John",
  last: "Doe",
  get fullName() {
    return this.first + " " + this.last;
  },
  set fullName(name) {
    [this.first, this.last] = name.split(" ");
  }
};
user.fullName;              // "John Doe" — getter called like property
user.fullName = "Jane Smith"; // setter called on assignment

Function Methods: call, apply, bind

const obj = { a: 5 };
function add(x, y) { return x + y + this.a; }

// .call(thisArg, arg1, arg2) — call with specific 'this'
add.call(obj, 3, 4);    // 3 + 4 + 5 = 12

// .apply(thisArg, [args]) — same but args as array
add.apply(obj, [3, 4]); // 12

// .bind(thisArg, partialArg) — returns NEW function with fixed 'this' / args
const add3 = add.bind(obj, 3); // pre-fills x=3
add3(4);   // 3 + 4 + 5 = 12

Prototypes — JS's Underlying OOP

Before ES6 classes, JS used prototype chains for inheritance. Each object can have a prototype object — it "inherits" all of that prototype's properties.

const base = { a: 1, inc() { this.a++; } };

// Create object with base as prototype
const child = Object.create(base);
child.a;         // 1 — inherited from prototype
child.inc();     // calls base.inc(), increments child.a
child.a;         // 2

ES6 Classes — Syntactic Sugar over Prototypes

class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    return this.name + " makes a sound";
  }
  get info() { return "Animal: " + this.name; }
}

class Dog extends Animal {
  constructor(name) {
    super(name);  // calls parent constructor
  }
  speak() {
    return this.name + " barks";  // overrides parent method
  }
}

const d = new Dog("Rex");
d.speak();  // "Rex barks"
d.info;     // "Animal: Rex" — inherited getter
Classes in JS are syntactic sugar — under the hood they use prototypes. The class keyword was added in ES6 for readability.

Asynchrony, Event Loop & Promises

The Call Stack

JS uses a call stack (LIFO) to track function execution. Each function call adds a stack frame. When a function returns, its frame is popped off.

function h() { return 42; }
function g() { return h(); }
function f() { return g(); }
f();
// Stack: [main → f → g → h] then unwinds back

The Event Loop

JS is single-threaded — it can only run one piece of code at a time. The event loop manages tasks:

  • Call Stack: currently executing code
  • Task Queue: pending tasks (timers, network responses, clicks)
  • Event loop: when the call stack is empty, picks the next task from the queue
Blocking: a long synchronous loop freezes the entire UI — no clicks respond, nothing renders. Never run long computations synchronously in the browser.

Callbacks

Pass a function to be called when an async operation completes.

// setTimeout — schedules callback after delay (goes to task queue)
setTimeout(() => {
  console.log("runs after 2 seconds");
}, 2000);
console.log("this runs FIRST");  // synchronous code runs immediately

// Callback pattern for async file read (Node.js)
fs.readFile('data.txt', (err, data) => {
  if (err) console.log(err);
  else console.log(data);
});
Callback hell: deeply nested callbacks become unreadable. Promises and async/await solve this.

Promises

const p = new Promise((resolve, reject) => {
  // async work...
  resolve("success");   // or reject(new Error("failed"))
});

p.then(result => console.log(result))
 .catch(err => console.log(err));

// Chain multiple async operations cleanly
fetch("/api/data")
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

async / await — Cleanest Syntax

async function getData() {
  try {
    const res = await fetch("/api/data"); // waits for promise
    const data = await res.json();        // waits again
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}
getData();
// async/await is syntactic sugar over Promises
// await can only be used INSIDE an async function

JSON — JavaScript Object Notation

What Is JSON?

A text-based format for transferring data between systems. Originally informal JS syntax; now standardized as RFC 7159 / ECMA-404. Used by virtually all REST APIs.

JSON vs JS Object Literal

FeatureJSONJS Object
KeysMust be double-quoted stringsAny valid identifier
StringsDouble quotes onlySingle or double quotes
Functions❌ Not allowed✅ Allowed
undefined❌ Not allowed✅ Allowed
Comments❌ Not allowed✅ Allowed
Trailing commas❌ Not allowed✅ Allowed

JSON.stringify & JSON.parse

const user = { name: "Yash", age: 20, active: true };

// JS object → JSON string
const json = JSON.stringify(user);
// '{"name":"Yash","age":20,"active":true}'

// Pretty-print with indentation
JSON.stringify(user, null, 2);

// JSON string → JS object
const parsed = JSON.parse(json);
parsed.name;  // "Yash"

// What gets dropped during stringify
const data = { fn: () => {}, x: undefined, y: 42 };
JSON.stringify(data);  // '{"y":42}' — fn and undefined dropped!

JSON in REST APIs

// Sending JSON to an API
await fetch("/api/users", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ name: "Yash", age: 20 })
});

// Receiving JSON from an API
const res = await fetch("/api/users");
const users = await res.json();  // auto-parses JSON
res.json() is shorthand for JSON.parse(await res.text()). It returns a Promise.

⚡ Key Takeaways for Assignments

  • Arrays are objects — typeof [] is "object". Use Array.isArray(x) to check.
  • map/filter/reduce/find — know what each does and which returns a new array vs a single value.
  • Default sort is string-based — always pass a comparator for numeric sort: (a,b) => a-b.
  • ES6 modules — named vs default export/import; only exported things are accessible.
  • Prototypes — JS's underlying OOP. ES6 class is syntactic sugar over prototypes.
  • call/apply/bind — all control what this is; bind returns a new function.
  • Event loop — JS is single-threaded; long sync code blocks the UI.
  • async/await — sugar over Promises; await only works inside async functions.
  • JSON.stringify drops functions and undefined — keys must be quoted strings in JSON.

🧠 Quick Revision Quiz

10 questions — click an option to check your answer.

Q1. What does [1,2,3].map(x => x * 2) return?

map transforms each element and returns a NEW array. It doesn't modify the original. find returns the first match, reduce accumulates to one value, filter returns matching elements.

Q2. What does [3,10,2].sort() return?

Default sort converts to strings: "10" < "2" < "3" alphabetically → [10, 2, 3]. Always use .sort((a,b) => a-b) for numeric sorting.

Q3. What is the difference between export default and export const?

export default fn → import fn from '...' (any name, no braces). export const x = 1 → import { x } from '...' (braces, exact name). A module can have many named exports but only one default.

Q4. What does const [a, b] = [10, 20, 30] do?

Array destructuring assigns elements left-to-right. Extra elements are silently ignored. You can use const [a, b, ...rest] = [10,20,30] to capture the rest.

Q5. In JS, classes are:

ES6 class syntax is just a cleaner way to write prototype-based inheritance. Under the hood, JS still uses the prototype chain. extends sets up the prototype relationship.

Q6. What is the JS event loop?

JS is single-threaded. The event loop monitors the call stack and task queue. When the stack is empty, it moves the next task from the queue to the stack. This is how async callbacks run without blocking.

Q7. What does fn.bind(obj, 5) return?

bind returns a NEW function — it doesn't call the original. The new function has this locked to obj and the first argument pre-filled to 5. call and apply call immediately; bind does not.

Q8. Which of these is NOT valid JSON?

In JSON, all keys must be double-quoted strings. age: 20 without quotes is JS object literal syntax, not valid JSON. All other options follow JSON rules correctly.

Q9. What happens to functions and undefined values when you JSON.stringify an object?

JSON.stringify({fn: ()=>{}, x: undefined, y: 42}) → '{"y":42}'. Functions and undefined are not valid JSON values, so stringify silently drops those keys entirely.

Q10. await can only be used:

await pauses the current async function until the promise resolves. It must be inside an async function. Using await in a regular function causes a SyntaxError. Modern JS also supports top-level await in ES modules.
