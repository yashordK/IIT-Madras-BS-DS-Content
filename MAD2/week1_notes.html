L2

JavaScript Origins & Overview

Birth of JavaScript (1995)

  • Sun Microsystems → Java applets (small programs inside browsers)
  • Netscape wanted similar interactivity → hired Brendan Eich
  • Created in roughly 10 days to bind HTML elements together
  • Named "JavaScript" to ride Java's popularity → Sun complained → ECMA standardized it as ECMAScript (ECMA-262)

Why JS Is "Forgiving"

HTML is forgiving (errors don't crash pages). JS was designed the same way — if it threw errors constantly, users would abandon it. This design choice causes quirks.

ECMAScript Versions

VersionKey Point
ES5Safe baseline — works everywhere
ES6 / ES2015Major update: let/const, arrow functions, classes, modules
ES2017+async/await, yearly releases
Course uses modern JS (ES6+) — don't worry about old compatibility unless told to.

Compatibility Tools

  • Polyfills: libraries that emulate newer features in older browsers
  • Transpilers (Babel): convert modern JS syntax to older syntax

ASI — Automatic Semicolon Insertion

JS doesn't require semicolons but tries to insert them automatically. Edge cases can cause bugs. Prof's rule: always use semicolons.

Running JS

// In browser: include in HTML
<script src="app.js"></script>
// Or inline
<script> console.log("hello") </script>

// Outside browser: Node.js
node filename.js

var vs let vs const — THE Most Important Difference

varletconst
ScopeFunction scopeBlock {}Block {}
ReassignYesYesNo
Use it?❌ AVOID✅ Use✅ Prefer
{
  let x = 10;
  var y = 20;
}
console.log(y);  // 20 — var leaks out of block!
console.log(x);  // ReferenceError — let stays inside block

const — What It Actually Means

const = the binding cannot be reassigned. The value can still be mutated.

const x = 5;
x++;         // TypeError — can't reassign const

const arr = [1, 2, 3];
arr.push(4);  // ✅ Fine — mutating array, not reassigning arr

const in for loops — Tricky!

// ✅ Works — each iteration creates a NEW binding
for (const x of [1,2,3]) { console.log(x); }

// ❌ Fails — i++ tries to REASSIGN const
for (const i = 0; i < 5; i++) { }  // TypeError!
for (let i = 0; i < 5; i++) { }   // ✅ Use let here

Control Flow & Functions

String — UTF-16 Length Quirk

JS strings are UTF-16. .length gives UTF-16 code units, not characters. For ASCII it's fine; emoji/some scripts may count as 2.

undefined vs null

let arr = [];
arr[5];       // undefined — slot exists but was never assigned

let x = null; // null — programmer explicitly chose "no value"

Loop Types

// Classic C-style
for (let i = 0; i < 5; i++) { console.log(i); }

// for...in — iterates KEYS/INDICES
const obj = {a: 1, b: 2};
for (const key in obj) { console.log(key); }  // "a", "b"

// for...of — iterates VALUES
const arr = [10, 20, 30];
for (const val of arr) { console.log(val); }  // 10, 20, 30
for...in = keys | for...of = values — easy to mix up!

3 Ways to Define Functions

// 1. Named Declaration (hoisted — can be called before definition)
function add(x, y) { return x + y; }

// 2. Anonymous Expression (NOT hoisted)
const greet = function() { return "hello"; };

// 3. Arrow Function (ES6) — best for short/callback functions
const square = x => x * x;               // 1 param: no parens needed
const add2 = (x, y) => x + y;           // implicit return, no braces
const bigger = (x, y) => {              // with braces: need explicit return
  const result = x + y;
  return result;
};
Arrow functions without {} have implicit return. With {}, you must write return explicitly.

DOM API

Why Not Just console.log?

console.log is for debugging only. JS was designed for document manipulation — reading inputs from the UI and updating what the user sees.

Core DOM Methods

// Select an element by ID
const el = document.getElementById("myId");

// Set HTML content (allows tags, ⚠️ XSS risk with user input)
el.innerHTML = "Hello <b>World</b>";

// Set plain text (safer)
el.innerText = "Hello World";

// Listen for events
el.addEventListener("click", function() {
  // runs on click
});

Click Counter Demo (from Replit in lecture)

<div id="d1">Click me</div>
<script>
  let x = 0;
  const d1 = document.getElementById("d1");
  d1.addEventListener("click", function() {
    x++;
    d1.innerHTML = "Click count: " + x;
  });
</script>

setTimeout — Preview of Async

// Runs after 2 seconds
setTimeout(() => {
  document.getElementById("d1").innerHTML = "Welcome!";
}, 2000);

⚡ Key Takeaways for Assignments

  • var vs let vs const — var is function-scoped (AVOID). let/const are block-scoped.
  • const in for loops — works in for...of (new binding), fails with i++ (reassignment).
  • typeof null === "object" — famous quirk, null is NOT an object.
  • undefined vs null — undefined = not yet set; null = programmer chose "no value".
  • for...in = keys, for...of = values — don't mix them up.
  • Arrow functions — no braces = implicit return; with braces = explicit return needed.
  • Semicolons — always use them, don't rely on ASI.
  • Comments — use // and /* */, NOT #.

🧠 Quick Revision Quiz

10 questions — click an option to check your answer.

Q1. What does typeof null return in JavaScript?

This is a famous JS quirk. typeof null === "object" — even though null is NOT an object. It's a historical bug that was never fixed for backwards compatibility.

Q2. Which of the following will cause a TypeError?

i++ tries to reassign const i — TypeError. In for...of loops, each iteration creates a new binding so const works. Pushing to an array or adding a property doesn't reassign the binding.

Q3. What is the difference between for...in and for...of?

for...in → keys/indices. for...of → values. On {a:1, b:2}: for...in gives "a","b". On [10,20]: for...of gives 10,20.

Q4. Which variable declaration has function scope (not block scope)?

var is function-scoped — it leaks out of {} blocks. let and const are block-scoped. Always avoid var.

Q5. What does this arrow function return: const f = x => x * 2; when called as f(5)?

Arrow functions without {} have an implicit return. x => x * 2 is equivalent to function(x) { return x * 2; }. So f(5) returns 10.

Q6. When was JavaScript created and who created it?

Brendan Eich created JS at Netscape in 1995 in about 10 days. ECMA later standardized it as ECMAScript to avoid Sun's Java trademark issues.

Q7. Which DOM method attaches an event handler to an element?

addEventListener is the standard DOM API. The others don't exist in modern browsers (attachEvent was IE-only, removed).

Q8. What does undefined mean vs null?

undefined: JS assigned this because you declared a variable but never gave it a value. null: you explicitly set it to "nothing". Both are falsy but semantically different.

Q9. What is a transpiler like Babel used for?

Transpilers like Babel take modern ES6+ syntax and output ES5 so older browsers can run it. Polyfills do something related but different — they add missing APIs as libraries.

Q10. Which function style allows the function to be called BEFORE its definition in the code?

Only named function declarations are hoisted — JS moves them to the top of the scope before execution. Expressions and arrow functions are NOT hoisted.
